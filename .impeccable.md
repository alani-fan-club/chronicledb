## Design Context

### Users
SillyTavern community — public OSS users who install ChronicleDB as a plugin. Range from casual RP hobbyists to power users running long multi-character campaigns. Technical enough to set up Postgres + a server plugin, but not necessarily developers. They open the mindmap in a browser tab alongside SillyTavern.

### Brand Personality
Curious, layered, unhurried. The tool is about discovering narrative connections you didn't notice while writing — an archaeological dig through your own stories, not a command center.

### Aesthetic Direction
Open to exploration. The current Shift5 (military/industrial) aesthetic is functional but not sacred. The overhaul should prioritize usability and polish over any specific style. Dark theme is appropriate given the context (RP sessions, often at night). Coral (#FF5841) accent is established but can evolve.

### Design Principles
1. **Discovery over surveillance** — the graph is for finding connections, not monitoring metrics. Favor exploration patterns over dashboard patterns.
2. **Characters are the stars** — character nodes and their relationships are the primary objects of interest. Everything else (events, arcs, facts) is supporting cast.
3. **Dense but navigable** — real graphs have hundreds of nodes. The UI must handle density without becoming unusable, but shouldn't hide complexity behind progressive disclosure that makes exploration tedious.
4. **Quiet chrome, vivid data** — UI controls should recede; the graph and its data should be the loudest thing on screen.
5. **Works at every zoom** — from the full-graph overview down to a single character's detail panel, every zoom level should feel intentional.
