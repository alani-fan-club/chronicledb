# Migrations

ChronicleDB does not use a formal migration framework. The plugin is redeployed in-place against existing live databases, so every schema change must be forward-compatible with older deployments. The canonical pattern is idempotent `CREATE TABLE IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`, and `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` statements in `server-plugin/schema.sql`, executed on plugin boot.

## Pattern

When evolving a table:

1. Add an `ALTER TABLE <name> ADD COLUMN IF NOT EXISTS <col> <type> [DEFAULT ...];` line to `schema.sql`, located immediately after the table's original `CREATE TABLE` block so the column appears on both fresh installs (via the CREATE) and existing installs (via the ALTER).
2. Keep the column nullable, or supply a `DEFAULT`, so the `ALTER` succeeds against rows already in the table.
3. If a backfill is required (e.g. recomputing embeddings), add a dedicated script under `server-plugin/` and document how to run it alongside the ALTER.
4. Never drop a column in place without first confirming no in-flight deployment still writes it. Prefer adding a replacement column and flipping readers, then removing the old column in a follow-up.

New tables follow the same rule: use `CREATE TABLE IF NOT EXISTS`. New indexes use `CREATE INDEX IF NOT EXISTS`. On plugin boot, `schema.sql` is executed top-to-bottom and any statement that is already applied is a no-op.

## Current post-initial-schema migrations

These are the ALTER / backfill pairs currently living in `schema.sql`:

- `events.source_text` ŌĆö `ALTER TABLE events ADD COLUMN IF NOT EXISTS source_text TEXT DEFAULT ''`. Distinctive 1ŌĆō2 sentence extract per event, populated by the extractor.
- `events.tier` ŌĆö `ALTER TABLE events ADD COLUMN IF NOT EXISTS tier TEXT DEFAULT 'recent'`. Lifecycle tier (`recent` / `condensed` / `archived`).
- `events.condensed_summary` ŌĆö `ALTER TABLE events ADD COLUMN IF NOT EXISTS condensed_summary TEXT`. Summary used after an event is condensed into longer-horizon storage.
- `events.embedding` ŌĆö `ALTER TABLE events ADD COLUMN IF NOT EXISTS embedding vector(768)`. Backfilled by `backfill-multi-granularity-embeddings.js`.
- `memory_embeddings.context_prefix` ŌĆö `ALTER TABLE memory_embeddings ADD COLUMN IF NOT EXISTS context_prefix TEXT`. Anthropic-style contextual retrieval blurb prepended at embed time. Backfilled by `backfill-context-prefix.js`.
- `memory_embeddings.raw_text` ŌĆö `ALTER TABLE memory_embeddings ADD COLUMN IF NOT EXISTS raw_text TEXT`. Unprefixed original text.
- `memory_embeddings.tsv` ŌĆö `ALTER TABLE memory_embeddings ADD COLUMN IF NOT EXISTS tsv tsvector GENERATED ALWAYS AS (to_tsvector('english', content)) STORED`. Powers lexical search.
- `context_snapshots.embedding` ŌĆö `ALTER TABLE context_snapshots ADD COLUMN IF NOT EXISTS embedding vector(768)`. Backfilled by `backfill-multi-granularity-embeddings.js`.
- `dialogue_quotes` ŌĆö entire table added post-initial-schema via `CREATE TABLE IF NOT EXISTS` and its accompanying GIN `tsv` / GIN `trgm` indexes. No backfill: older chats remain without dialogue quotes until re-ingested.
- `items.chat_id` ŌĆö `ALTER TABLE items ADD COLUMN IF NOT EXISTS chat_id TEXT`. Scopes `OWNS` / `LOCATED_AT` edges to a single chat. Content-addressed id now includes `chat_id` so the same-named item in two chats becomes two rows. Legacy NULL-chat rows are dropped from chat-scoped queries; re-ingest to populate.
- `knows.chat_id` ŌĆö `ALTER TABLE knows ADD COLUMN IF NOT EXISTS chat_id TEXT`. Scopes `KNOWS` edges so a character can know a fact in chat A without leaking it into chat B. Unique constraint stays `(character_id, fact_id)`; `chat_id` is a metadata tag, first-write-wins via `COALESCE`. Legacy NULL-chat rows dropped under scoped reads.
- `present_at.chat_id` ŌĆö `ALTER TABLE present_at ADD COLUMN IF NOT EXISTS chat_id TEXT`. Scopes character presence to a chat so extraction in chat A doesn't flip `is_current=FALSE` on rows from chat B.
- `traits.normalized_content` ŌĆö generated `STORED` column that lowercases `content` and strips non-alphanumeric. Backed by a UNIQUE index on `(character_id, category, normalized_content)` so `upsertTrait` dedupes case/punctuation variants ("Awed" / "awed" / "Awe-struck" / "Awestruck") via `ON CONFLICT DO NOTHING`. One-time cleanup deletes pre-existing duplicates keeping the oldest row per normalized group.
- `traits.stemmed_content` ŌĆö generated `STORED` column from `strip(to_tsvector('english', content))::text`. Runs the Postgres English stemmer so `charmed` / `charming` / `charms` all collapse to `'charm'`, `observant` / `observing` to `'observ'`, etc. `upsertTrait`'s fuzzy pre-check compares against this to catch morphological variants the exact-normalized index misses. Non-unique index (not UNIQUE) so legitimate multi-word traits with overlapping stems can coexist. One-time cleanup kept the longest content per `(character, category, stemmed_content)` group.
- `traits.embedding` ŌĆö `ALTER TABLE traits ADD COLUMN IF NOT EXISTS embedding vector(768)`. Per-trait contextual embedding populated lazily by the extractor (Path 1 contextual embeddings). Nullable: older rows stay NULL until re-ingested. Feeds the per-character rollup in `characters.summary_embedding` ŌĆö `recomputeCharacterSummary` skips rows where `embedding IS NULL`.
- `characters.summary_embedding` ŌĆö `ALTER TABLE characters ADD COLUMN IF NOT EXISTS summary_embedding vector(768)`. Per-character mean-pool rollup of all `personality`-category trait embeddings, computed via pgvector's native `AVG(vector)`. Recomputed by `db.recomputeCharacterSummary` on trait insert/delete; backfilled across all characters via `POST /api/plugins/chronicle-db/recompute-character-summaries`. Unlocks "find similar characters across chats" as a retrieval primitive (research report ┬¦5 Path 4).
- `idx_characters_summary_hnsw` ŌĆö `CREATE INDEX IF NOT EXISTS idx_characters_summary_hnsw ON characters USING hnsw (summary_embedding vector_cosine_ops)`. HNSW cosine index backing character-similarity kNN queries against the rollup embedding.
- `traits.evidence_sentence` ŌĆö `ALTER TABLE traits ADD COLUMN IF NOT EXISTS evidence_sentence TEXT`. One-sentence verbatim quote from the source batch that justifies the trait; becomes the tail half of the contextual embedding text (`"${character} is ${content}: ${evidence_sentence}"`) per Anthropic's contextual retrieval pattern. Nullable ŌĆö legacy rows embed as `"${character} is ${content}"` alone.
- `traits.canonical_id` ŌĆö `ALTER TABLE traits ADD COLUMN IF NOT EXISTS canonical_id TEXT REFERENCES traits(id) ON DELETE SET NULL`. Self-referential pointer to the canonical trait row when this row is a merged variant; `NULL` means this row IS the canonical. All user-visible trait reads filter `WHERE canonical_id IS NULL` so merged variants never surface twice. On canonical deletion the reference nulls out rather than cascading so the raw-extraction provenance survives.
- `traits.aliases` ŌĆö `ALTER TABLE traits ADD COLUMN IF NOT EXISTS aliases TEXT[] DEFAULT '{}'`. Array of raw extraction strings merged into this canonical row (each element looks like `"content: evidence"` or just `"content"` when no evidence). Lets retrieval do full-text fallback on merged synonyms without a second table.
- `traits.merged_count` ŌĆö `ALTER TABLE traits ADD COLUMN IF NOT EXISTS merged_count INT DEFAULT 1`. Number of extraction candidates (including the original) that have been folded into this canonical row. Cheap popularity signal for ranking.
- `idx_traits_canonical` ŌĆö `CREATE INDEX IF NOT EXISTS idx_traits_canonical ON traits (character_id, canonical_id)`. Speeds up the "get all merged variants for a canonical id" lookup and the user-visible `WHERE canonical_id IS NULL` filter per character.
- `idx_traits_embedding_hnsw` ŌĆö `CREATE INDEX IF NOT EXISTS idx_traits_embedding_hnsw ON traits USING hnsw (embedding vector_cosine_ops) WHERE canonical_id IS NULL`. Partial HNSW cosine index restricted to canonical rows. `upsertTrait`'s kNN lookup uses this index to find the nearest existing canonical trait for the same character + category; merged alias rows are intentionally excluded so the kNN never returns a row that's already been collapsed. Backfill and /recompute-character-summaries both depend on this index being populated for merged-pass to do anything.
- `traits.verified_at` ŌĆö `ALTER TABLE traits ADD COLUMN IF NOT EXISTS verified_at TIMESTAMPTZ`. Stamped by the Path 3 LLM verifier (research report ┬¦5 Path 3) every time a candidate in the 0.80ŌĆō0.88 cosine band is evaluated against this canonical, regardless of the MERGE / KEEP_DISTINCT / REJECT_NEW outcome. Observability signal for "which canonicals have been tuned by the verifier" and rough staleness tracking. Prospective only ŌĆö no backfill; legacy canonicals stay NULL until a new candidate lands in their verify band.
