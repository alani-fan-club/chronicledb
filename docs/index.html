What ChronicleDB Does [overview]

Every message you exchange gets analyzed by a lightweight LLM. Characters, events, traits, relationships, locations, items, and plot threads are extracted into a PostgreSQL graph with vector embeddings for semantic search. When it's time for the AI to reply, the most relevant memories are retrieved and injected into context — so the model knows what happened, who said what, and how characters feel about each other.

SillyTavern Chat → Extraction LLM → Graph + Vector Store AI Generation ← Hybrid Retrieval (RRF) ← PostgreSQL + pgvector

Installation Guide [5 steps]

The install script automates everything. It detects your platform, installs PostgreSQL + pgvector if needed, clones the repo, symlinks into SillyTavern, and creates the database. You'll be chatting with memory in under 5 minutes.

Prerequisites

  • macOS or Linux (Windows users: use WSL2)
  • SillyTavern installed and working
  • Node.js 18 or newer
  • A Gemini or OpenAI API key (free tier works)

PostgreSQL and pgvector are installed automatically if missing. You don't need to set them up beforehand.

Step by Step

Run the installer

One command. It will ask where your SillyTavern lives if it can't auto-detect it.

curl -fsSL https://raw.githubusercontent.com/alani-fan-club/chronicledb/master/install.sh | bash

The script is idempotent — safe to re-run if anything goes wrong. It installs PostgreSQL via Homebrew (macOS) or apt (Linux), enables pgvector and pg_trgm extensions, creates the chronicledb database, and symlinks the plugin into SillyTavern.

Restart SillyTavern

Stop and start SillyTavern so it picks up the new plugin. You should see this in the console:

[ChronicleDB] Initializing server plugin...
[ChronicleDB] Mind map path: /path/to/chronicledb/src/ui
[ChronicleDB] Server plugin ready.

If you don't see these lines, check that enableServerPlugins: true is set in your SillyTavern config.yaml. The installer does this automatically, but a config file merge during an ST update could revert it.

Open the settings panel

In SillyTavern, scroll down in the Extensions sidebar and expand the ChronicleDB drawer. You'll see the connection status, database settings, and API configuration.

The database fields should be pre-filled with the defaults from the installer (localhost:5432, database chronicledb). Click Connect & initialize to verify the connection.

If you see a username mismatch warning ("Connected as X but configured Y"), Postgres trust auth is accepting any username. Either fix pg_hba.conf to require password auth, or update the username to match what Postgres logged you in as.

Configure your LLM

Under Memory model, paste your API key and select a model. Under Embedding model, do the same (or leave the key blank to reuse the extraction key).

FieldRecommended Value
API TypeGemini (free tier is fine)
API KeyYour Gemini or OpenAI key
Extraction Modelgemini-2.5-flash-lite
Embedding Modelgemini-embedding-2-preview
Embedding Dimension768 (default, don't change)

Don't use expensive models for extraction. A $0.01/M-token model is plenty — the extraction prompt is structured and doesn't need frontier reasoning. Save your budget for the actual roleplay model.

Start chatting

That's it. With Auto-build memory enabled (on by default), ChronicleDB starts extracting and storing memories as you chat. You'll see a toast notification on each new chat confirming memory is active.

Under Behavior, you can tune:

Common Questions [faq]
Does this send my chats to an external API?
The extraction and embedding models process your chat text. If you use Gemini or OpenAI, that text goes to their API. If you run Ollama or LM Studio locally, everything stays on your machine. The PostgreSQL database is always local. ChronicleDB never phones home — it's a SillyTavern plugin that talks only to the endpoints you configure.
How much does it cost?
Gemini's free tier handles most usage. Flash Lite extraction costs ~$0.01 per 1M tokens. Gemini embeddings are free. If you use OpenAI, text-embedding-3-small is $0.02 per 1M tokens. A typical chat session costs less than a penny.
Does it work on Windows?
The installer targets macOS and Linux. On Windows, install WSL2 (Ubuntu), then run the install script inside WSL. SillyTavern itself runs on Windows, but the plugin's server side (PostgreSQL + pgvector) needs a Unix environment.
What happens when I swipe or regenerate?
The padding window (default: 5 messages) keeps recent turns out of the graph entirely. If you swipe or edit within that window, nothing needs cleanup. For older messages that were already ingested, ChronicleDB detects the swipe and surgically deletes the old extraction before re-extracting the new content.
Can I use a cloud database instead of local PostgreSQL?
Yes. Any PostgreSQL 14+ instance with pgvector works. Neon, Supabase, and Railway all support pgvector out of the box. Just update the host, port, database, user, and password fields in the settings panel. The installer's auto-setup is local-only, but the plugin doesn't care where the database lives.
How do I import existing chats?
Open the Character memory section in settings, select a character, and click Build memory from all. ChronicleDB reads every past chat file for that character and batch-processes them through the extraction pipeline. Depending on chat length and your API speed, this can take a few minutes.
