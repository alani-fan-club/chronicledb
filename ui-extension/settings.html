Almost there — one manual step

ChronicleDB needs SillyTavern's server plugins enabled. Add this one line to your config.yaml, then restart SillyTavern:

enableServerPlugins: true

Likely locations for config.yaml:

  • Windows: %USERPROFILE%\SillyTavern\config.yaml (or under SillyTavern-Launcher)
  • macOS: ~/SillyTavern/config.yaml
  • Linux: ~/SillyTavern/config.yaml

Database

Memory is stored in an embedded PGlite database under ~/.chronicledb/pgdata. No setup, no Postgres install — click Connect & initialize below to provision it on first run.

Memory model

LLM that reads your chats and builds the memory. Any OpenAI-compatible or Gemini endpoint works. A cheap, fast model is fine.

Embedding model

Converts memories into vectors for semantic search. Gemini, Vertex AI (Express mode), and any OpenAI-compatible endpoint are supported (OpenAI proper, Mistral, Voyage, Ollama, LM Studio, vLLM, OpenRouter, LiteLLM, ...). The schema is fixed at 768 dimensions — pick a 768-dim model, or one that supports a dimensions parameter (OpenAI's text-embedding-3-small / 3-large and Vertex's text-embedding-004 / 005 do).

Fixed at 768 — the schema's vector(768) columns are baked into the indexes. Pick an embedding model that natively outputs 768, or one that accepts a dimensions parameter (OpenAI 3-small / 3-large, Vertex 004 / 005, Voyage) and the plugin will truncate. Setting any other value here would only desync embed-time from store-time and break inserts.

Behavior

How memory gets built while you chat.

When on, every reply automatically updates memory for the current chat — no manual ingest needed. Turn off if you only want memory from chats you explicitly ingest.

Recent messages stay in ST's native context until they're N turns behind. Skipping ingest on the latest N means swiping or editing recent replies doesn't need cleanup because nothing was written to the graph yet. Default 5.

Recomputes the story-arc structure (super-arcs / arcs / episodes) on the auto-ingest path every N new events. Cluster titles are recycled when their centerpiece event is unchanged, so the per-fire LLM cost is near zero on stable partitions. 0 disables — only the manual full-chat ingest will rebuild arcs.
