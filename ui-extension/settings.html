PostgreSQL Connection

Extraction LLM

Reads RP messages and extracts structured memories (characters, events, relationships, plot threads). Any OpenAI-compatible or Gemini endpoint.

Context LLM

Cheap model that writes a 1-2 sentence "situating blurb" for each ingested message — who, where, what's happening — prepended to the text before embedding so search hits carry scene context. Shares the Extraction API URL, key and type. Leave blank to reuse the extraction model.

Embedding Model

Generates vector embeddings for semantic search. Currently Gemini-only (server plugin calls the Gemini embed endpoint directly).

The embedder reuses the Extraction API key (or the legacy Gemini API key if set). 768 matches the current memory_embeddings.embedding column width — changing dimension requires a schema migration.

Behavior

Retrieval is hybrid: vector similarity + full-text matched events + matched dialogue quotes + recent snapshots + plot threads, all scored together and truncated to the budget above.

Memory Types

Which categories the memory block surfaces. Extraction still writes all types — these toggles only affect retrieval display.
